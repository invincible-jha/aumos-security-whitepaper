# Security Policy

## Reporting a Vulnerability

MuVeraAI takes security vulnerabilities seriously. If you discover a security vulnerability in the AumOS platform or this documentation, please follow responsible disclosure.

### Contact

**Email:** security@muveraai.com
**PGP Key:** Available on request
**Response SLA:** 24 hours for initial acknowledgment, 72 hours for triage

### Do Not

- Open a public GitHub issue for security vulnerabilities
- Disclose publicly before we have had a chance to remediate
- Exploit or test against production systems without written authorization

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Any proof-of-concept (kept confidential)

## Supported Versions

| Version | Security Support |
|---------|-----------------|
| 1.x (current) | Full support |
| < 1.0 | Not supported |

## Disclosure Timeline

| Stage | Timeline |
|-------|----------|
| Initial acknowledgment | 24 hours |
| Triage and severity classification | 72 hours |
| Remediation plan communicated | 7 days |
| Fix delivered (critical/high) | 30 days |
| Fix delivered (medium/low) | 90 days |
| Public disclosure (coordinated) | After fix + 30 days |

## Bug Bounty

MuVeraAI operates a private bug bounty program. Qualifying researchers are invited by invitation. Contact security@muveraai.com to inquire.
